Method: MessageManager::detailsActive()

public function detailsActive() : bool;

Returns the state whether the display of the message details is active or not.

Details

If the display of the message details is active, they are shown in the frontend.

The message details are generally managed over the setting show_prog_warning. But you can also temporarily disable message details, e.g. to avoid displaying of unnecessary warnings when you make a check that can produce a programming warning, but it is a controlled noticed and should not bother the user.

Returns

bool

Returns the state whether the display of the message details is active or not.

protected function extractMessagesForDisplay(array &$messages) : array;

This is internal auxiliary function for preparing messags for displaying.

Details

When the messages are requested, it is assumed they will be displayed. Thus, the message array is cleared to avoid displaying of the same messages twice.

The message details are removed if the flag show_message_details is false.
